E2E Dashboard · UX + API + Gap

GoPop E2E Dashboard

Flow lengkap dari sudut pandang tim internal — siapa login, lihat apa, aksi apa, endpoint apa yang dipanggil, dan gap apa yang masih kurang.

D-A1
Login Admin Dashboard
Entry point semua role — Super Admin, Finance, Ops, CS, IT
⚙️ Irwan + Rifky (BE)⚙️ Isal + Ipong (FE Dashboard)
Happy Path Admin login berhasil → redirect ke dashboard sesuai role ▾
UX Flow
  • 1
    Buka gopop-lilac.vercel.app
    Redirect otomatis ke /login jika belum ada session
  • 2
    Input email + password
    Tidak pakai nomor HP — email based untuk admin
  • 3
    Login berhasil → masuk dashboard
    Sidebar menu muncul sesuai role. Super Admin lihat semua menu. CS hanya lihat Komplain, Pengguna, Monitoring (read-only).
API
POST /admin/auth/login → return access_token + refresh_token + role + permissions[]

Response:
{ "access_token": "...", "refresh_token": "...", "admin": { "id": "ADM-001", "nama": "Acoy", "role": "super_admin", "permissions": ["transaksi.read","transaksi.write","refund.approve",...] } }
Dashboard State
Frontend simpan token + role di localStorage. Sidebar render menu berdasarkan permissions[]. Setiap navigasi ke menu sensitif → re-check permission dari token.
Gap yang Ditemukan
D-GAP-01
Login dashboard belum ada di repo — masih hardcode / mock
File login.html di repo ada tapi tidak terintegrasi dengan BE auth. Tidak ada POST /admin/auth/login endpoint yang terdokumentasi.
✅ Fix: Buat endpoint login admin terpisah dari consumer auth. Admin pakai email, consumer pakai nomor HP.
Owner: Irwan + Rifky (BE) · Isal + Ipong (FE)
D-GAP-02
Tidak ada session management / auto-logout
Dashboard tidak ada mekanisme session timeout. Kalau admin lupa logout, session tetap aktif selamanya — security risk.
✅ Fix: Access token 8 jam, refresh token 7 hari. Auto-logout setelah idle 2 jam. Interceptor 401 → redirect login.
Owner: Irwan + Rifky (BE) · Isal + Ipong (FE)
E1 Password salah / akun tidak ditemukan ▾
UX
Pesan: "Email atau kata sandi salah." Setelah 5x salah → akun terkunci 30 menit. Super Admin bisa unlock dari Role & Akses.
API
POST /admin/auth/login → 401 INVALID_CREDENTIALS dengan field remaining_attempts. Setelah 5x → 423 ACCOUNT_LOCKED + retry_after.
