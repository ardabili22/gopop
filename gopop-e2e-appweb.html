AW-A
Auth Flow — Login, Register, Lupa Password
Figma: Section "Login" sudah ada · OTP screen belum ada
⚙️ Irwan + Rifky (BE)⚙️ Agung (App)⚙️ Fikly (Desktop)
Happy Path Register baru → OTP → Login berhasil ▾
UX — Step by Step
  • 1
    Tap "Daftar" dari Home atau soft prompt post-transaksi
  • 2
    Form Register — Nama + Nomor HP + Password
    Validasi real-time: format HP, kekuatan password (min 8 char + angka). Tombol submit disabled sampai semua valid.
  • 3
    OTP dikirim via WA → screen OTP
    6 kotak OTP + countdown 5 menit + tombol "Kirim Ulang" (aktif setelah 60 detik). ❌ Screen ini belum ada di Figma
  • 4
    OTP benar → akun aktif → redirect Home (sudah login)
API
EndpointMethodBody / Response
/auth/registerPOSTBody: {name, phone, password} → Return: {user_id, status: "UNVERIFIED"}
/auth/verify-otpPOSTBody: {phone, otp} → Return: {access_token, refresh_token, user}
/auth/resend-otpPOSTBody: {phone} → Rate limit: max 3x, cooldown 60 detik
/auth/loginPOSTBody: {phone, password} → Return: {access_token, refresh_token, user}
/auth/refreshPOSTBody: {refresh_token} → Return: new access_token
/auth/forgot-passwordPOSTBody: {phone} → Return: 200 selalu (cegah enumeration)
/auth/reset-passwordPOSTBody: {phone, otp, new_password} → Invalidate semua session
Gap
AW-GAP-01
Screen OTP Verification belum ada di Figma
Register dan Forgot Password butuh screen OTP input. Tanpa screen ini Agung dan Fikly tidak bisa implement auth flow.
✅ Action Bili: Buat screen OTP di Figma — 6 input box + countdown timer + tombol Kirim Ulang + error state.
Owner: Bili (design) → Agung + Fikly (implement)
AW-GAP-02
Tidak ada interceptor refresh token di FE
Kalau access token expired saat user lagi di app, semua API call akan return 401. Tanpa interceptor, user tiba-tiba keluar dari app tanpa penjelasan.
✅ Fix: HTTP interceptor di Agung (App) dan Fikly (Desktop): saat 401 → auto hit /auth/refresh → retry request. Kalau refresh juga 401 → redirect login dengan pesan kontekstual.
Owner: Agung (App) + Fikly (Desktop)

UX — Step by Step
  • 1
    Pilih produk dari Homepage atau Layanan Semua
    Transfer E-Wallet: screen 01 · Pulsa: screen 06 · Voucher Game: screen 14-16 · Token Listrik: screen 19-20
  • 2
    Input nomor tujuan / ID
    Validasi format real-time. Pulsa: detect operator otomatis dari prefix. Token Listrik: input ID Pelanggan.
  • 3
    Screen 03 — Checkout: Pilih Metode Bayar
    Tampil: detail produk, harga, pilihan QRIS / VA / E-Wallet / Retail.
  • 4

    Seharusnya tampil kode bayar + countdown timer + instruksi per metode. Ini gap kritis.

  • 6

API
EndpointMethodKeterangan
/productsGETQuery: ?category=pulsa&operator=telkomsel. Public.
/products/prefix-detectGETQuery: ?phone=0812xxxx → return operator name. Public.
/transactionsPOSTBody: {product_id, destination, source_channel}. Auth opsional (guest bisa). Return: {transaction_id, payment_instruction}
/transactions/{id}GETPolling status. Return: {status, product_detail, ...}
/payments/callbackPOSTWebhook dari Xendit. Internal — tidak dipanggil FE.
Gap
AW-GAP-06
Payment Instruction screen belum ada di Figma
Screen yang muncul setelah user pilih metode bayar (QRIS, VA, E-Wallet) tidak ada di Figma. Ini screen paling sering dilihat user saat transaksi — yang menampilkan kode bayar dan countdown timer.
✅ Action Bili: Buat 3 variant — QRIS (tampil QR code + countdown), VA (tampil nomor VA per bank + instruksi transfer), E-Wallet (deeplink redirect + fallback "Buka aplikasi X").
Owner: Bili (design) → Agung + Fikly (implement)
AW-GAP-07
Polling status tidak ada mekanisme stop
FE perlu polling GET /transactions/{id} tiap 5 detik tapi tidak ada definisi: kapan stop polling? Kalau user tutup app saat polling, apakah ada background check?
✅ Fix: Stop polling saat status bukan PENDING/PAID/PROCESSING, atau setelah timeout 15 menit. Setelah 10 menit tampil pesan reassurance. Tidak perlu background — kalau app ditutup, update datang via WA notif.
Owner: Agung (App) + Fikly (Desktop) — logic FE
AW-GAP-08
source_channel tidak dikirim dari FE
POST /transactions harus menyertakan source_channel tapi ini belum didefinisikan di contract antara Agung (App), Fikly (Desktop), dan Irwan (BE).
✅ Fix: Agung kirim source_channel: "app_consumer" atau "app_guest". Fikly kirim "web_consumer" atau "web_guest". Tergantung ada tidaknya auth token.
Owner: Agung (App) + Fikly (Desktop) — 1 baris kode per FE
